Hubsport API v1 — Endpoints nuevos

Base URL de ejemplo: https://yordy.shop/v1
Cabeceras habituales: Accept: application/json · Rutas protegidas: Authorization: Bearer <token>

PDF: en el navegador, menú Archivo → Imprimir → Destino “Guardar como PDF”.

1. Stories — /stories

Listar stories

MétodoGET
Ruta/v1/stories
AuthBearer (Sanctum)
Query / BodyNinguno
200{ "data": { "<user_ulid>": [ …stories… ], … } } (agrupado por usuario; stories activas, tuyas + seguidos)

Crear story

MétodoPOST
Ruta/v1/stories
AuthBearer
Content-Typeapplication/json
Body JSON
  • media_url (string, requerido) — URL válida del medio
  • type (string, requerido) — image o video
201{ "message", "data": { story } } — expira ~24 h
422Validación

2. Connections (solicitudes) — /connections

Listar conexiones aceptadas

MétodoGET
Ruta/v1/connections
AuthBearer
200{ "message", "data": [ ConnectionResource ] }

Solicitudes entrantes (pendientes)

MétodoGET
Ruta/v1/connections/requests/incoming
AuthBearer
200Lista con sender cargado (UserResource)

Solicitudes salientes (pendientes)

MétodoGET
Ruta/v1/connections/requests/outgoing
AuthBearer
200Lista con recipient cargado

Enviar solicitud

MétodoPOST
Ruta/v1/connections
AuthBearer
Body JSON{ "connected_user_id": "<ULID usuario>" } — debe existir en users.id
201 / 200Solicitud creada o reabierta desde rejected
409Ya conectados, ya pendiente, o el otro ya te envió solicitud (connection_id)
422Contigo mismo u otras validaciones

Aceptar solicitud

MétodoPOST
Ruta/v1/connections/{connection}/accept
Pathconnection = id numérico (tabla connections)
AuthBearer — solo el destinatario (connected_user_id)
200Estado accepted
403 / 422No autorizado o ya no está pending

Rechazar solicitud

MétodoPOST
Ruta/v1/connections/{connection}/reject
AuthBearer — solo destinatario
200Estado rejected

Eliminar / cancelar

MétodoDELETE
Ruta/v1/connections/{connection}
AuthBearer — debe participar en la conexión
200Cancelar saliente, descartar entrante, unfriend si accepted, borrar rejected

ConnectionResource (resumen): id, status (pending|accepted|rejected), fechas, y opcionalmente sender / recipient (UserResource).

3. Meta — ayuda y contacto

Ayuda / FAQ (estructura)

MétodoGET
Ruta/v1/help
AuthNo
200{ "message", "data": { "sections": [ { "id", "title", "articles": [] } ] } }

Formulario de contacto

MétodoPOST
Ruta/v1/contact
AuthOpcional. Sin sesión: email obligatorio.
Body JSON
  • subject (requerido, max 200)
  • message (requerido, max 5000)
  • email — requerido sin token; opcional con token
202Mensaje recibido (registro en log del servidor)
